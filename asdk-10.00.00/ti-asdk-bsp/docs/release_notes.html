Introduction

Audio SDK provides a cohasive platform for audio signal chain processing and its integration with DSP Concept's AWECore. It includes mcu+ sdk for AM62Dx device, low-level drivers and AVB stack, This document also will provide a list of new features supported in this release.

What's New

10.00.00 (EA) Release

  • NEW FEATURES
    • This is Audio SDK beta-release (10.00.00). It supports multi-channels audio procesing on C7x and A53.
    • The release supports audio signal-chain in multiple configurations
      1. MCU/R5F + C7x : C7x is running at 1 GHz
      2. MCU/R5F + A53 : A53 is running at 1.4 GHz
      3. MCU/R5F + C7x + A53 (Multi-Instance): C7x is runningat 1 GHz, A53 is configured as coprocessor. Single layout with some algorithms are running on C7x and some modules are running on A53 concurrently. C7x is configured as primary core, owns ADC/DAC and A53 is configured as secondary core.
    • Multi-channels onboard audio: 8-channels of ADC input and 8-channels of DAC output at 48 kHz is supported in this release.
    • Ethernet Audio: 8-channels single stream AVB (Audio Video Bridge) audio as listener and talker
    • Supports multirate/multithread layout : Single layout is broken into multiple sub-layout (low-latency mode + normal latency mode). Low latency sublayout runs on higher priority thread than normal priority thread.
    • MCU/R5F is used as tuning interface, also runs AVB stack
    • Audio algorithms are highly optimized for C7x architecture. The release includes following additional kernel optimized for C7x.
      • FIR
      • FIRNChannel
      • FIRDecimator
      • FIRInterpolator
      • ScaleSmoothNCh
      • AGCLimiterCore_Nch
      • Matrix multiply
      • Db20, Db10
      • ScalarControl
      • Correlation
      • TableInterp2d
      • TableLookup
      • fast sine, fast cos, fast atan
      • Log, Log2, Log10
      • Pow, Pow10toX
      • Substract2x1, Divide2x1
      • Xor2x1
      • Magnitude
      • Min
  • KNOWN LIMITATIONS
    • Smoothing on coefficients for BiquadSparseV6 and SecondOrderFilterSmoothedCascadeV2 Module not working as expected. Set the smoothing coefficient to 0 for now
    • Number of active biquad stages in each channel for BiquadSparseV6 Module cannot be more than the initial count. i.e. If the module has 2 channels with 8 stages per channel with first 4 stages are active and remaining 4 stages are bypassed per channel, it won't work if any of the bypassed stages in any channel are made active during runtime. i.e. max stages are locked to initial active count
