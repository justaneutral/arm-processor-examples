DSPLIB_qrd

Introduction

Kernel for QR decomposition of input matrix.

  • This function decomposes the real square matrix \(A\) into an orthogonal matrix and an upper right triangular matrix as:

    \begin{eqnarray*} A = Q \times R \qquad where\;&(1)&\;Q\;is\;an\;orthogonal\;matrix\;(i.e. Q^{T}Q=I)\\ &(2)&\;R\;is\;upper\;triangular\;matrix. \end{eqnarray*}

  • The Householder algorithm is used to zero the columns below the diagonal in the original \(A\) matrix to produce the \(R\) matrix.
  • The \(Q\) matrix is generated during the process.

Memory Requirements
Buffer dimY dimX Comments
pA M M Input Square Matrix
pQ M M Output Square Matrix
pR M M Output Square Matrix
pU 1 M Scrath buffer to store intermediate values
pScratch 1 M Scratch buffer to store intermediate sum values
Returns
Status value indicating success or failure. Refer to DSPLIB_STATUS.
Assumptions:
  • None
Performance Considerations:
For best performance,
  • The input and output data buffers are expected to be in L2 memory
  • The buffer pointers are assumed to be 64-byte aligned
  • Ensure that the starting point of adjacent rows in a 2-D matrix are allocated from different banks in L2 memory with the help of memory padding.
Remarks
Before calling this function, application is expected to call DSPLIB_qrd_init and DSPLIB_qrd_exec_checkParams functions. This ensures resource configuration and error checks are done only once for several invocations of this function.
