◆ VXLIB_thresholdBinary_init_ci()

template<uint32_t dTypeIn, uint32_t dTypeOut>
VXLIB_STATUS VXLIB_thresholdBinary_init_ci ( VXLIB_kernelHandle  handle,
const VXLIB_bufParams2D_t *  bufParamsIn,
const VXLIB_bufParams2D_t *  bufParamsOut,
const VXLIB_thresholdBinary_InitArgs *  pKerInitArgs 
)

This function is the initialization function for the C7x implementation of the kernel. The function declaration conforms to the declaration of VXLIB_thresholdBinary_init.

This function determines the configuration for the streaming engine resources based on the function call parameters, and the configuration is saved in bufPBlock array. In the kernel call sequence, VXLIB_thresholdBinary_exec_ci would be called later independently by the application. When VXLIB_thresholdBinary_exec_ci runs, it merely retrieves the configuration from the bufPBlock and uses it to set up the hardware resources. This arrangement is so that VXLIB_thresholdBinary_exec_ci does not lose cycles to determine the hardware configuration.

Parameters
[in]handle: Active handle to the kernel
[in]bufParamsIn: Pointer to the structure containing dimensional information of input image
[out]bufParamsOut: Pointer to the structure containing dimensional information of output image
[in]pKerInitArgs: Pointer to the structure holding init parameters
Returns
Status value indicating success or failure. Refer to VXLIB_STATUS.

template<typename dTypeIn , typename dTypeOut >
VXLIB_STATUS VXLIB_thresholdBinary_exec_ci ( VXLIB_kernelHandle  handle,
void *restrict  pIn,
void *restrict  pOut,
void *restrict  pThresholdVal,
void *restrict  pTrueVal,
void *restrict  pFalseVal 
)

This function is the main execution function for the C7x implementation of the kernel. The function declaration conforms to the declaration of VXLIB_thresholdBinary_exec.

Parameters
[in]handle: Active handle to the kernel
[in]pIn: Pointer to buffer holding the input image
[out]pOut: Pointer to buffer holding the output data
[in]pThresholdVal: Pointer to the threshold value
[in]pTrueVal: Pointer to the true value
[in]pFalseVal: Pointer to the false value
Returns
Status value indicating success or failure. Refer to VXLIB_STATUS.
Performance Considerations:
For best performance,
  • the input and output data images are expected to be in L2 memory
  • the buffer pointers are assumed to be 64-byte aligned
