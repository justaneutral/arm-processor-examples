VXLIB_histogram
VXLIB kernels

Introduction

Kernel for creating a histogram distribution from an image based on a given given offset, range, and number of bins.

Generates a histogram distribution from an image. This kernel counts the number of occurrences of each pixel value within the window size of a pre-calculated number of bins. The following datatype combinations are supported:

Case Input Output
0 uint8_t uint32_t
Method
  if ((offset <= pixel) && (pixel < (offset + range))) {
         index = ((pixel - offset) * numBins) / range;
         histogramBin[index]++;
   }
Assumptions
  • Scratch buffer is assumed to have 1024 entries
  • numBins should be <= 256
  • (offset + range) should be <= 256
  • All dist and scratch entries should be externally initialized to zero before calling the function for the first time since it is always read by the function regardless if the function is being called once per image, or multiple times per image
  • The dist output of the function is only considered accurate for the whole image when the lastBlock parameter is set to 1.

)

This function is the main kernel compute function.

Please refer to details under VXLIB_histogram_exec

Parameters
[in]handle: Active handle to the kernel
[in]pIn: Pointer to the structure input image
[out]pOut: Pointer to the output image
[out]pTemp: Pointer to the temporary scratch buffer for histogram
Returns
Status value indicating success or failure. Refer to VXLIB_STATUS.
Assumptions:
  • pTemp scratch buffer assumed to have 1024 entries
Performance Considerations:
For best performance,
  • the input and output data buffers are expected to be in L2 memory
  • the buffer pointers are assumed to be 64-byte aligned
Remarks
Before calling this function, application is expected to call VXLIB_histogram_init and VXLIB_histogram_exec_checkParams functions. This ensures resource configuration and error checks are done only once for several invocations of this function.
