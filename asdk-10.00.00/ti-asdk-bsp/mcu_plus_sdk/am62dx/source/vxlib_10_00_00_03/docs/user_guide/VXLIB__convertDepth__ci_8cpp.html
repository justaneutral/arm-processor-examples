template<uint32_t dTypeIn0, uint32_t dTypeOut>
VXLIB_STATUS VXLIB_convertDepth_init_ci ( VXLIB_kernelHandle  handle,
const VXLIB_bufParams2D_t *  bufParamsIn0,
const VXLIB_bufParams2D_t *  bufParamsOut,
const VXLIB_convertDepth_InitArgs *  pKerInitArgs 
)

This function is the initialization function for the C7x implementation of the kernel. The function declaration conforms to the declaration of VXLIB_convertDepth_init.

This function determines the configuration for the streaming engine and MMA hardware resources based on the function call parameters, and the configuration is saved in bufPBlock array. In the kernel call sequence, VXLIB_convertDepth_exec_ci would be called later independently by the application. When VXLIB_convertDepth_exec_ci runs, it merely retrieves the configuration from the bufPBlock and uses it to set up the hardware resources. This arrangement is so that VXLIB_convertDepth_exec_ci does not lose cycles to determine the hardware configuration.

Parameters
[in]handle: Active handle to the kernel
[in]bufParamsIn0: Pointer to the structure containing dimensional information of input image
[out]bufParamsOut: Pointer to the structure containing dimensional information of ouput image
[in]pKerInitArgs: Pointer to the structure holding init parameters
Returns
Status value indicating success or failure. Refer to VXLIB_STATUS.

◆ VXLIB_convertDepth_exec_ci()

template<typename dTypeIn0 , typename dTypeOut >
VXLIB_STATUS VXLIB_convertDepth_exec_ci ( VXLIB_kernelHandle  handle,
void *restrict  pIn0,
void *restrict  pOut 
)

This function is the main execution function for the C7x implementation of the kernel. The function declaration conforms to the declaration of VXLIB_convertDepth_exec.

The function uses MMA hardware accelerator to perform the convolution computation. Filter data is loaded into B panel of the MMA from memory using one streaming engine, while the input data is loaded into A vectors of the MMA using the other streaming engine. Result of the compute from MMA C panel is stored into memory using a stream address generator.

Parameters
[in]handle: Active handle to the kernel
[in]pIn0: Pointer to buffer holding the input image 0
[out]pOut: Pointer to buffer holding the output image
Returns
Status value indicating success or failure. Refer to VXLIB_STATUS.
Performance Considerations:
For best performance,
  • the input and output data buffers are expected to be in L2 memory
  • the buffer pointers are assumed to be 64-byte aligned
