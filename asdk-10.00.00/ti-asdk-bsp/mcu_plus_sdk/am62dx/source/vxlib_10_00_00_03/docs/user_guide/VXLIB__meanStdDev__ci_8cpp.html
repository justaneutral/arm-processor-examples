template<uint32_t dTypeIn, uint32_t dTypeOut>
VXLIB_STATUS VXLIB_meanStdDev_init_ci ( VXLIB_kernelHandle  handle,
const VXLIB_bufParams2D_t *  bufParamsIn,
const VXLIB_meanStdDev_InitArgs *  pKerInitArgs 
)

This function is the initialization function for the C7x implementation of the kernel. The function declaration conforms to the declaration of VXLIB_meanStdDev_init.

This function determines the configuration for the streaming engine resources based on the function call parameters, and the configuration is saved in bufPBlock array. In the kernel call sequence, VXLIB_meanStdDev_exec_ci would be called later independently by the application. When VXLIB_meanStdDev_exec_ci runs, it merely retrieves the configuration from the bufPBlock and uses it to set up the hardware resources. This arrangement is so that VXLIB_meanStdDev_exec_ci does not lose cycles to determine the hardware configuration.

Parameters
[in]handle: Active handle to the kernel
[in]bufParamsIn: Pointer to the structure containing dimensional information of input image
[in]pKerInitArgs: Pointer to the structure holding init parameters
Returns
Status value indicating success or failure. Refer to VXLIB_STATUS.

This function is the main execution function for the C7x implementation of the kernel. The function declaration conforms to the declaration of VXLIB_meanStdDev_exec.

Parameters
[in]handle: Active handle to the kernel
[in]pIn: Pointer to buffer holding the input image
[in,out]pPixelsProcessed: Pointer to buffer holding number of pixels processed
[in,out]pCurrentSum: Pointer to buffer holding current pixels sum
[in,out]pCurrentSqSum: Pointer to buffer holding current pixels squared sum
[out]pOut: Pointer to buffer holding the output data
Returns
Status value indicating success or failure. Refer to VXLIB_STATUS.
Performance Considerations:
For best performance,
  • the input and output data images are expected to be in L2 memory
  • the buffer pointers are assumed to be 64-byte aligned
